Universal Rich Presence Fix

Enable Discord RPC for ANY Flatpak application (VS Code, Minecraft, etc.). The ultimate guide to breaking out of the sandbox.

‚ö†Ô∏è How It Works

The Problem: Flatpak applications are sandboxed and cannot see the "socket" that Discord uses to communicate.

To fix this, we need to ensure the Host (Discord/Vesktop) creates the socket where standard apps expect it, and the Client (Your App) has permission to touch it.

üõ†Ô∏è Troubleshooting Steps

Step 1: Symlink Fix (Global Fix)

This creates a permanent link so the Host exposes the socket globally.

1
Run Setup Commands

Open your terminal and run this to link the Host path to the standard path:

$ln -sf $XDG_RUNTIME_DIR/{app/com.discordapp.Discord,}/discord-ipc-0 $mkdir -p ~/.config/user-tmpfiles.d $echo 'L %t/discord-ipc-0 - - - - app/com.discordapp.Discord/discord-ipc-0' > ~/.config/user-tmpfiles.d/discord-rpc.conf $systemctl --user enable --now systemd-tmpfiles-setup.service
2
Restart Your Device

A full restart is required for the systemd changes to properly take effect.

Step 2: Flatseal Permissions

If Step 1 didn't work, you need to explicitly grant filesystem permissions using Flatseal.

A
For The Host (Discord/Vesktop)

The host app needs access to both its own folder (to create sockets) and the shared path.

xdg-run/app/com.discordapp.Discord:create xdg-run/discord-ipc-0:create
B
For The Client (Your App)

Your target app (e.g., VS Code, Prism Launcher, Anki) only needs access to the socket:

xdg-run/discord-ipc-0

Step 3: Advanced Debugging

Still not working? Let's dig deeper to find the root cause.

1
Reset & Test

Try removing the existing IPC file to clear any bad states:

$rm $XDG_RUNTIME_DIR/discord-ipc-0
2
Run from Terminal

Launch both apps from the terminal to see live logs. Replace com.example.App with your app's ID.

$flatpak run com.example.App $flatpak run com.discordapp.Discord

Look for permission denied errors regarding xdg-run or discord-ipc-0.
